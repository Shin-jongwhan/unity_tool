# ProceduralTree 제작 방법

## 핵심 원리
참고 이미지의 **실루엣**(갈색 줄기 + 위로 갈수록 작아지는 원뿔 스커트 N단)을 **정점 단위로 코드 생성**.
폴리곤 예산을 낮게 잡아 모바일 친화(현재 **62 트라이앵글**). 색은 **코드로 그린 세로 그라데이션 텍스처**를
각 단 내부에 매핑 → 단별 음영으로 입체감.

> 이 접근은 "참고 이미지 + 폴리곤 예산 → 코드로 로우폴리 메시 생성" 방식이다.
> (AI 에셋 생성기/외부 모델 임포트 없이 순수 코드.)

## 구조
- **줄기**: 살짝 가늘어지는 6각 기둥, 갈색 단색.
- **수관**: `Tiers`개의 원뿔을 높이를 겹쳐 쌓아 층층 스커트 실루엣.
  - 각 단 i: 바닥 링(`Seg`개 정점, 반경 `r`) + 꼭짓점 1개로 원뿔 측면.
  - 반경 `BottomR`(아래) → `TopR`(위) 선형 감소.
  - 양면 렌더(`_Cull=0`)로 와인딩 무관.
- **UV.v**: 단 바닥 0.12 / 꼭짓점 0.95 → 그라데이션이 단마다 "아래 진녹 / 위 연녹".
- **그라데이션 텍스처**: 16×64 코드 생성(진녹→중간→연녹 + 펄린 노이즈).

## 파라미터(상수)
| 상수 | 기본 | 의미 |
|---|---|---|
| `Seg` | 10 | 원뿔 둘레 분할(낮을수록 각짐) |
| `Tiers` | 5 | 스커트 단 수(많을수록 촘촘·뾰족) |
| `TrunkH` | 0.7 | 줄기 높이 |
| `CanopyH` | 3.3 | 수관 높이 |
| `BottomR / TopR` | 1.15 / 0.10 | 맨 아래/위 단 반경 |

## 변형 아이디어
- 더 뾰족: `Tiers`↑, `BottomR`↓
- 둥근 활엽수: 원뿔 대신 구/반구
- 설원: 그라데이션에 흰색 혼합 또는 단 위 흰 캡 추가
